# FX ドル円アラートシステム 設計書

Issue #51 の壁打ち（2026-07-22）で確定した設計の単一の真実源。
戦略ルールの本体は [docs/strategy/fx-usdjpy-technical-manual.md](../strategy/fx-usdjpy-technical-manual.md) を参照。

## 目的

「エントリーチャンスの見逃し」の解消。裁量トレードのルール優位性は実績で確認済み扱いであり、
課題は判定精度ではなく**監視の網羅性**。システムはルールベースで市況を常時監視し、
セットアップ候補を Slack に通知する。**エントリーの最終判断は人間の裁量**（LLM による
エントリー判定は不採用と決定済み・再提案しない）。

## 3段階ロードマップ

| Phase | 内容 | 取引方法 |
|---|---|---|
| 1 | テキストのみの Slack 通知（read-only） | スマホのブローカーアプリで手動 |
| 2 | チャート画像添付 + Slack リアクション承認 → 発注（Slack App + Socket Mode） | Slack 上で承認 |
| 3 | 完全自動売買 | 事後報告通知のみ |

Phase 1 から判定結果を**構造化データ**で持ち、Phase 2 の発注パラメータ・Phase 3 の執行に流用する。
チャート画像は Phase 1 では不要（判定は数値のみ）。Phase 2 の添付から必要になり、自前描画
（mplfinance）を推奨で仮合意。

## 全体構成（Phase 1）

```
[価格データ取得層] → [指標計算 (Layer 0)] → [判定エンジン (Layer 1-2)] → [通知整形 (Layer 3)] → Slack
        ↑                                          ↑
  GMO コイン為替 FX public API              経済指標カレンダー
  （OANDA v20 API に差し替え予定）          （Forex Factory 週間 JSON・キャッシュ）
```

- **可変ポーリング**: 上位足で環境認識 → 条件接近時に下位足の監視頻度を上げる
- **監視タイムフレーム**: 日足・4時間足・1時間足・15分足の4枚。1分足は自動化対象外。
  5分足（戦略2のトリガー足）は Phase 1 では見送り、必要になったら追加（保留事項）
- **チャート閲覧**: 人間は TradingView を見る（取引自動化 = Phase 3 までは MT5 不使用。
  発注は OANDA API 想定のため MT5 は当面設計から外す）

## データ取得層

- **GMO コイン為替 FX public API で作り始め、OANDA v20 REST API に差し替える**
  （OANDA 口座は再開設手続き中のため）。取得層を抽象化し、差し替えをインターフェースの裏に閉じ込める
- 経済指標カレンダーも同じ思想で抽象化する（詳細:
  [docs/research/economic-calendar-source.md](../research/economic-calendar-source.md)）

## 指標定義（TradingView の実設定に一致させる・実機スクショで確認済み）

| 指標 | 設定 | 実装上の注意 |
|---|---|---|
| EMA | 20 / 50 / 200 | SMA ではない |
| MA 傾き | 8本前との単純比較 | フラット誤判定が目立ったら変化率閾値方式に切替 |
| BB | 期間 20・**ベースMA = EMA**・ソース終値・±2σ と ±3σ の2枚 | ライブラリのデフォルトは SMA 基準。EMA 明示必須（ズレ要因） |
| SQZMOM | LazyBear 版（SQZMOM_LB）全デフォルト: BB 20/2.0・KC 20/1.5・TrueRange ON | |
| 直近高安 | スイング検出（フラクタル方式・左右5本） | 固定 lookback は不採用 |
| SMC | BigBeluga - Smart Money Concepts 1.0.0 相当（Swing 100・OB 直近5個・Mitigation=終値・Sweep ON） | プロプライエタリのため完全一致は狙わず `smart-money-concepts` ライブラリ + 「終値抜けで OB 無効化・直近5個追跡・スイープ検出」で近似 |

±2σ/±3σ の使い分け: 2σ タッチ = 過熱の初期サイン、2σ〜3σ 間の推移 = バンドウォーク
（逆張り禁止）、3σ タッチ = 統計的な行き過ぎ・利確検討（ユーザー確認済み）。

ブーストの「15m 80MA〜200MA の深い押し」は 1h の 20EMA〜50EMA ゾーンと同一のため、
15m に 80MA は追加せず **1h の EMA から直接計算**する。

## 判定エンジン 3 層構造

### Layer 0: 環境認識（全タイムフレームで常時計算）

EMA 20/50/200 の値・傾き（8本前比較）・パーフェクトオーダー（PO）判定 / BB 位置3値
（バンド内・2σ超・3σ超）・スクイーズ/エクスパンション・バンドウォーク / SQZMOM
（squeeze on/off・モメンタム方向・拡大縮小）/ 直近高安（フラクタル）/ SMC OB・スイープ。

### Layer 1: 触らない条件ゲート（1つでも該当したら見送り扱い）

マニュアル §5 + Gem 制御ルール由来。見送りでも判定内訳は通知に含める（沈黙しない）。

| # | 条件 | 数値判定 |
|---|---|---|
| 1 | 200EMA 外側の逆張り | 価格位置 vs 15m 200EMA |
| 2 | 50/200 の向き不一致・横ばい | 8本前比較の符号不一致 |
| 3 | 形状の乱れ | 実体/レンジ比 < 閾値が N 本連続（閾値はキャリブレーション対象） |
| 4 | 重要指標前・週またぎ | 指標60分前は新規禁止（30分前は決済リマインドに切替）・金曜クローズ前 |
| 5 | BB スクイーズ中は戦略1見送り | SQZMOM squeeze on |
| 6 | 週足 EMA 帯との衝突 | 週足 EMA 20/50/200 ± バッファ内 |
| 7 | 上位足トレンド終焉懸念の節目 | v1 は厳密判定せず「1h/4h 直近高安・OB 接近」を warning 表示 |
| 8 | RR < 1.5R | SL/TP 候補の機械算出で判定。**2R 確保可否を併記**（ユーザー決定） |

データ取得失敗・指標計算不能時は判定せず「判定不能」を通知する（推測で補完しない）。

### Layer 2: セットアップ検知（イベント駆動）

通知トリガーは一致度 % の閾値ではなく **MA タッチイベント**（マニュアル §3 のアラート定義に準拠）:

- **20EMA タッチ = 準備通知（早期警報）** — 環境確認の合図
- **50EMA ゾーンタッチ = 検討通知（警報）** — チェックリスト評価を添付

戦略判定:

- **戦略1（15m MA プルバック）**: 前提 = 200EMA 順張り側 + 50/200 同方向。モード判定:
  - 初動: `200EMA(now) > 200EMA(8本前) かつ 200EMA(8本前) ≤ 200EMA(16本前)`（下降は逆）。
    ロット 1/2〜1/3・1トレンド1回。再トライ条件は Phase 1 では通知文に列挙して人間判断
  - ブースト: 1h PO 確定時。深押し許容（1h 20-50EMA ゾーン）・TP は 1h 高安・
    出口ルール（押し安値割れ / 20EMA 実体2本連続割れ / 1h 包み足逆行）を継続監視して通知
  - 通常: 上記以外
- **戦略2（裏タッチ・ロールリバーサル）**: ① 1h 構造確定（ダウ HH-HL/LL-LH or ネックライン
  抜け → フラクタルスイングから機械判定・実装難度中）+ ② 節目裏側への回帰 & 15m EMA が
  節目±バッファ内に重合。SQZMOM 方向一致必須
- **ハイブリッド（先行+本命）**: Phase 1 は「1h 節目の 15m 実体ブレイク + SQZMOM 強拡大」の
  検知を通知に含めるのみ（先行玉の判断は人間）
- **トリガー形状（ヒゲ反発・包み足・W/三尊）**: Phase 1 は候補形状の検出まで。最終判断は人間

### Layer 3: 通知内容

- チェックリスト内訳（マニュアル §3.2 の4項目: MA 向き一致 / 200EMA 位置 / SQZMOM 一致 /
  指標なし + 戦略・モード該当）と一致度 %
- SL/TP 候補・RR（1.5R ゲート・2R 確保可否併記）
- ロット比率（フル / 1/2 / 1/3 等。マニュアル §4.1 の比率定義に基づく）
- 見送り時はその理由（どのゲートに該当したか）

## Slack 連携

- Phase 1: Incoming Webhook（送信のみ）
- Phase 2: Slack App + Socket Mode（リアクション承認 → 発注。誤タップ防止と承認期限が必要）

## 保留事項

- 5分足のポーリング追加（戦略2 トリガー足。Phase 1 は 15m のみで開始）
- 連敗時ロット制限（§4.2）は取引履歴が必要なため Phase 1 は人間管理。Phase 2 以降で履歴連携
- 形状の乱れ・週足 EMA バッファ等の閾値は過去チャートでのキャリブレーション前提
