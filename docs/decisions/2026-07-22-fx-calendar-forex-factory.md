# 決定の要約

経済指標カレンダーのデータソースは Forex Factory 週間 JSON フィードを採用する。

- **決定日:** 2026-07-22
- **記録者:** Claude（ユーザー回答に基づく）
- **参照:** tan3159/trade-scripts#51（論点5 壁打ち記録）・docs/research/economic-calendar-source.md

## 論点

指標60分前ゲート / 30分前リマインドに使う「発表時刻 + 重要度」を機械取得するソースの選定。

## 提示した選択肢

| 案 | 内容 |
|---|---|
| A（推奨） | Forex Factory 週間カレンダー JSON フィード — 無料・重要度付き・デファクト。非公式で仕様変更リスク |
| B | investing.com 等のスクレイピング — 規約グレー・壊れやすい |
| C | Trading Economics 等の商用 API — 公式・安定だが有料 |

## ユーザーの決定

**A**。「人間は TV（TradingView）見てるけど。機械的にとれるのは A がよさそうだね」。

## 理由・背景

用途が「時刻と重要度のみ・週1取得で足りる」と軽いため、無料・認証不要のフィードで十分。
人間向けの閲覧は TradingView のカレンダーを使い続ける（機械用と人間用を分離）。

## 今後 AI が取るべき行動

1. 実装は `https://nfs.faireconomy.media/ff_calendar_thisweek.json` を週1取得 + ローカルキャッシュ
2. 非公式フィードのため取得層を抽象化し、取得失敗・スキーマ不一致時は「カレンダー取得不能」を通知する
3. 商用 API への乗り換え提案はフィードが実際に壊れたときのみ行う
