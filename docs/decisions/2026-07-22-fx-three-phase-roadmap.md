# 決定の要約

FX アラートシステムは3段階で発展させる: Phase 1 = テキストのみ通知（取引は手動）、
Phase 2 = 画像添付 + Slack リアクション承認 → 発注、Phase 3 = 完全自動売買（事後報告のみ）。

- **決定日:** 2026-07-22
- **記録者:** Claude（ユーザー提案の追認・整理）
- **参照:** tan3159/trade-scripts#51（ロードマップコメント）

## 論点

チャート画像の要否と取得方式を議論する中で、システムの最終形と段階の切り方をどうするか。

## 提示した選択肢

ユーザー自身が3段階案を提示し、Claude が権限昇格モデル（read-only → 承認付き発注 → 自動執行）
として整理・追認した。対案なし。

## ユーザーの決定

3段階ロードマップを採用。「最初は通知が来てスマホでブローカーのアプリを立ち上げて取引。
次の段階は Slack 通知に画像添付、Slack 上でリアクションして取引指示。3段階目は完全取引自動化で
通知だけ来る」。

## 理由・背景

- Phase 1 ではチャートはスマホアプリ（TradingView / ブローカーアプリ）で見るため画像不要
- 判定は数値のみで行う方針（画像は判定に使わない）と整合
- いきなり自動売買にせず、通知精度の検証 → 承認付き発注 → 自動化と信頼を段階的に積む

## 今後 AI が取るべき行動

1. Phase 1 から判定結果を構造化データで持ち、Phase 2 の発注パラメータ・Phase 3 の執行に流用できる設計にする
2. チャート画像生成（mplfinance 想定）は Phase 2 まで実装しない
3. Slack は Phase 1 = Incoming Webhook、Phase 2 = Slack App + Socket Mode（誤タップ防止・承認期限が必要）
